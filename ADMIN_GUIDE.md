# 👑 Admin Panel Guide (EduVillage LMS)

This guide explains how administrators can use the Django Admin Dashboard (`/admin/`) to manage content, users, payments, and the system settings of EduVillage.

## 🚀 1. Accessing the Admin Panel
1. Go to `yourwebsite.com/admin/` or `localhost:8000/admin/`
2. Login with your Superuser credentials.
   *(If you don't have one, run `python manage.py createsuperuser` in the terminal)*

---

## 👩‍🎓 2. User & Subscription Management (ইউজার ও সাবস্ক্রিপশন মেইনটেনেন্স)
### `Users > Profiles`
Here you can see all registered students.
- **Subscription Tier:** You can manually upgrade a student's `subscription_tier` to `FREE`, `PRO` (100 Qs/day), or `PREMIUM` (500 Qs/day).
- **Api Usage:** You can reset a student's daily usage limit if they request it.

---

## 💰 3. Payment Approval (ম্যানুয়াল পেমেন্ট ভেরিফিকেশন)
### `Payments > Manual Payments`
If a student pays via **bKash, Nagad, or Rocket**, they will submit a form.
1. Match the **Transaction ID** provided by the student with your mobile banking SMS.
2. If the payment is valid, open the record and change the **Status** to `Approved`.
3. The system will **automatically upgrade** the user's `Profile` subscription tier based on the amount they paid.
4. If invalid, reject the payment.

---

## 📝 4. Managing the Blog (ব্লগ ও আর্টিকেল পোস্ট)
### `Blog > Categories`
- Always create a category first (e.g., "IELTS Tips", "SSC Suggestion").
- This categorizes content so students can filter easily on the frontend.

### `Blog > Posts`
- Click **Add Post** to write a new article.
- **Title:** The headline of the post.
- **Slug:** This is auto-generated (the URL piece).
- **Content:** Use the **CKEditor Text Editor** to write your blog. You can bold text, add lists, embed images, and design your post visually.
- **Featured Image:** Upload a thumb-nail (It requires `Pillow` to be installed).
- **Status:** Set to `Published` to make it live, or `Draft` if you are still writing it.

---

## 📚 5. IELTS Module Administration (IELTS মডিউল আপডেট)
### `IELTS > IELTS Reading Passages`
- Here you paste the reading texts. Add the `Title` and the `Content`.

### `IELTS > IELTS Questions`
- When you add questions, link them to a specific `Passage`.
- Support multiple types: `TRUE_FALSE_NOT_GIVEN`, `MULTIPLE_CHOICE`, `FILL_BLANK`, `MATCHING_HEADINGS`.
- **Very Important:** When creating Fill in the Blanks, ensure the correct answer exactly matches what the student is supposed to type.

---

## 🤖 6. AI Question Bank (প্রশ্ন ভাণ্ডার)
### `Questions > Question Banks`
- The system automatically saves all high-quality questions generated by the AI here.
- **Why check this?** If you notice the AI generated a wrong answer or bad formatting, you can edit it here. Next time the database is queried, it will show the correct version.
- **Manual Input:** You can also manually add hard/specific questions here. The AI fallback system will prioritize these.

---

## 🛠️ System Requirements Check (সার্ভার রিকোয়ারমেন্টস)
Make sure your production server or hosting has these components exactly installed via `pip install -r requirements.txt`:

```text
django==5.x
django-allauth (For sign up/login)
django-crispy-forms & crispy-bootstrap5 (For beautiful UI forms)
stripe (For credit card payments)
django-ckeditor (For blog rich-text editing)
pillow (Important: For saving blog and profile images)
psycopg2-binary (Crucial for Postgres database on Render/Railway)
```

**⚠️ Important Admin Rules:**
1. Never delete a `QuestionBank` item unless it is complete garbage, as students might have `Exam` logs tied to it.
2. Always test the `Stripe Webhooks` in test mode before going live with the payment gateway.
3. Keep the `DEBUG=False` in `settings.py` when your site is live to ensure hackers cannot see your errors or API Keys.
